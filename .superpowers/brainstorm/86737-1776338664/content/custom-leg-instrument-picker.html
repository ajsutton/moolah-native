Custom Leg Instrument Picker: UI Approaches

How should the instrument picker appear on each sub-transaction leg?

A

Inline Currency Picker (replace label)

The static currency label next to the amount (e.g. "AUD") becomes a tappable picker. Selecting an account pre-fills it, but the user can override. Compact — no extra rows.

Sub-transaction 1 of 2
Type Transfer
Account US Checking
Amount 100.00 USD ▾
Category None

Pros

  • No extra row — compact
  • Visually obvious that currency is editable
  • Amount and currency stay together

Cons

  • Tap target is small on iOS
  • Less discoverable — user might not realize it's tappable
B

Separate Instrument Row

Add a dedicated "Instrument" picker row to each sub-transaction, between Account and Amount. It defaults to the account's instrument and can be overridden. Standard Picker control — familiar pattern.

Sub-transaction 1 of 2
Type Transfer
Account US Checking
Instrument USD ▾
Amount 100.00 USD
Category None

Pros

  • Full-size tap target, standard Picker widget
  • Highly discoverable
  • Consistent with other picker rows (Account, Type, etc.)

Cons

  • Adds a row to every sub-transaction — more vertical space
  • Redundant with the currency shown next to Amount

Recommendation: Option B — it's consistent with the form's existing picker-row pattern (Type, Account, Category, Earmark are all separate rows). The added vertical space is acceptable since custom mode is already the "power user" view.
