# Shared Instrument Registry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move `InstrumentRecord`, the discovered-tokens inbox, and the price-cache tables out of per-profile storage and onto the existing `profile-index` zone and DB so spam decisions, discovered-token resolutions, and price-cache rows all propagate across profiles on the same iCloud account.

**Architecture:** Reuse the existing iCloud-account-scoped `profile-index` zone and `profile-index.sqlite` GRDB DB. Extend `ProfileIndexSyncHandler` to dispatch by `recordType`; add a new `SharedInstrumentScope` holder owned at app level; split `CryptoTokenStore` into a shared registry-data store and a per-session UI-state store; one-shot data-union migration walks every per-profile DB and merges into the shared tables. All work ships in one binary (the design's "single-binary atomicity" requirement).

**Tech Stack:** Swift 6, Swift Testing (`@Suite` / `@Test` / `#expect` / `#require`), GRDB 7, CKSyncEngine, Xcode project generated by `xcodegen` (`just generate`), test runner `just test` (filters: `just test SuiteName` / `just test SuiteName/methodName`).

**Companion spec:** `plans/2026-05-09-shared-instrument-registry-design.md` (already on `main` from PR #833). Read it before starting — every section here is grounded in a section there. The migration ID `v10_drop_shared_instrument_legacy` is reserved by an inline comment in `Backends/GRDB/ProfileSchema.swift` (already merged) for the follow-up cleanup spec; don't reuse that ID.

**Worktree / branch:** `.worktrees/shared-instrument-registry-plan`, branch `plan/shared-instrument-registry`. Already created from `origin/main` with `--no-track`. Use `git -C <path>` everywhere; never `cd && git`.

**Conventions reminders (from CLAUDE.md):**
- Tests are **Swift Testing**, not XCTest (`@Suite`, `@Test`, `#expect`, `await #expect(throws:)`).
- One protocol conformance per extension.
- Run `just format` before every commit. Don't touch `.swiftlint-baseline.yml`.
- Capture test output to `.agent-tmp/`. Delete temp files when done.
- TODOs require a GitHub issue reference; this plan adds none, so don't introduce any.
- `Instrument.Kind` raw values are `'fiatCurrency'`, `'stock'`, `'cryptoToken'` — not `'fiat'` / `'crypto'`. The CHECK constraint in the new schema must use the raw values.
- **All six rate-cache tables are `STRICT, WITHOUT ROWID`** in the per-profile schema's final form: the three body tables (`exchange_rate`, `crypto_price`, `stock_price`) have been `WITHOUT ROWID` since `v1_initial` (`Backends/GRDB/ProfileSchema+RateCaches.swift`); the three meta tables (`exchange_rate_meta`, `stock_ticker_meta`, `crypto_token_meta`) became `WITHOUT ROWID` at `v4_rate_cache_without_rowid` (`Backends/GRDB/ProfileSchema+RateCacheWithoutRowid.swift`). The companion design document on main says body tables are ROWID — that text is incorrect; this plan supersedes it. **Every writer of these tables must call `db.notifyRateCacheChange(...)` inside the same `db.write { }` block** because `sqlite3_update_hook` does not fire for `WITHOUT ROWID` tables (`Backends/GRDB/Observation/RateCacheTable.swift`).
- Test directories: tests live in `MoolahTests/Backends/CloudKit/` (not `MoolahTests/Backends/CloudKit/`), `MoolahTests/Backends/GRDB/`, `MoolahTests/App/`, `MoolahTests/Features/Settings/`, `MoolahTests/Migration/`. There is no `Sync` subdirectory under `Backends/CloudKit/` in the test target.
- `ProfileIndexDatabase.openInMemory()` already exists (`Backends/GRDB/ProfileIndexDatabase.swift`); use it for any test that needs a migrated in-memory profile-index DB. After this work it will create the new tables automatically.

// Backends/GRDB/ProfileIndexSchema+SharedInstrumentRegistry.swift

import Foundation
import GRDB

/// Body of the `v3_shared_instrument_registry` migration. Creates the
/// `instrument` table (mirrors the post-v8 per-profile shape) plus the
/// six rate-cache tables (mirrors the post-v4 per-profile shape).
///
/// **Retention policy.** The six rate-cache tables (`exchange_rate`,
/// `exchange_rate_meta`, `stock_price`, `stock_ticker_meta`,
/// `crypto_price`, `crypto_token_meta`) are **kept forever** — needed
/// for historic-conversion correctness on reports older than the
/// upstream rate APIs can serve. See `guides/DATABASE_SCHEMA_GUIDE.md`
/// §9 and the equivalent rationale in `ProfileSchema.swift:48-55`.
///
/// **Verbatim semantics.** Each `CREATE TABLE` reflects the table's
/// current (post-v8 for instrument; post-v4 for rate caches) shape in a
/// single statement; the historical per-profile evolution is not
/// replayed here — fresh-table creation does not need it.
extension ProfileIndexSchema {
  static func createSharedInstrumentRegistryTables(_ database: Database) throws {
    try database.execute(
      sql: """
        CREATE TABLE instrument (
            id                          TEXT    NOT NULL PRIMARY KEY,
            record_name                 TEXT    NOT NULL UNIQUE,
            kind                        TEXT    NOT NULL
                CHECK (kind IN ('fiatCurrency', 'stock', 'cryptoToken')),
            name                        TEXT    NOT NULL,
            ticker                      TEXT,
            exchange                    TEXT,
            coingecko_id                TEXT,
            binance_symbol              TEXT,
            cryptocompare_symbol        TEXT,
            contract_address            TEXT,
            chain_id                    INTEGER,
            decimals                    INTEGER NOT NULL CHECK (decimals >= 0),
            pricing_status              TEXT    NOT NULL DEFAULT 'priced'
                CHECK (pricing_status IN ('priced', 'unpriced', 'spam')),
            encoded_system_fields       BLOB
        ) STRICT;
        """)

    try createSharedRateCacheTables(database)
  }

  private static func createSharedRateCacheTables(_ database: Database) throws {
    // sqlite3_update_hook does not fire for WITHOUT ROWID tables. Any
    // caller writing into these tables (CryptoPriceService,
    // StockPriceService, ExchangeRateService persistence paths) MUST
    // call `db.notifyRateCacheChange(...)` inside the same db.write { }
    // block so RateCacheTickStream's ValueObservation re-fires.
    // (Same contract as the per-profile schema; see
    // ProfileSchema+RateCaches.swift for the original justification.)
    //
    // Note: the per-profile `exchange_rate_lookup` index on
    // (base, quote, date) is intentionally omitted from the shared
    // schema. No production query uses (base, quote, date) as SQL
    // predicates — loadCache issues `WHERE base = ?` (PK leading-column
    // scan) and resolves quote/date lookups in-memory from
    // caches[base]. Carrying the index forward would add write
    // amplification on a write-heavy table for zero read benefit. The
    // RateQueryPlanTests extension in Task 18 confirms the PK is the
    // chosen access path.
    try database.execute(
      sql: """
        CREATE TABLE exchange_rate (
            base TEXT NOT NULL,
            quote TEXT NOT NULL,
            date TEXT NOT NULL,
            rate REAL NOT NULL,
            PRIMARY KEY (base, date, quote)
        ) STRICT, WITHOUT ROWID;

        CREATE TABLE exchange_rate_meta (
            base TEXT PRIMARY KEY,
            earliest_date TEXT NOT NULL,
            latest_date TEXT NOT NULL
        ) STRICT, WITHOUT ROWID;

        CREATE TABLE stock_price (
            ticker TEXT NOT NULL,
            date TEXT NOT NULL,
            price REAL NOT NULL,
            PRIMARY KEY (ticker, date)
        ) STRICT, WITHOUT ROWID;

        CREATE TABLE stock_ticker_meta (
            ticker TEXT PRIMARY KEY,
            instrument_id TEXT NOT NULL,
            earliest_date TEXT NOT NULL,
            latest_date TEXT NOT NULL
        ) STRICT, WITHOUT ROWID;

        CREATE TABLE crypto_price (
            token_id TEXT NOT NULL,
            date TEXT NOT NULL,
            price_usd REAL NOT NULL,
            PRIMARY KEY (token_id, date)
        ) STRICT, WITHOUT ROWID;

        CREATE TABLE crypto_token_meta (
            token_id TEXT PRIMARY KEY,
            symbol TEXT NOT NULL,
            earliest_date TEXT NOT NULL,
            latest_date TEXT NOT NULL
        ) STRICT, WITHOUT ROWID;
        """)
  }
}
```

**Authoritative DDL source.** The skeleton above is illustrative for the structure of the migration body. The implementer must transcribe the exact column types, defaults, and CHECK constraints from the per-profile DDL — `ProfileSchema+CoreFinancialGraph.swift` for `instrument` (then apply the v8 `pricing_status` column from `ProfileSchema+CryptoWalletFields.swift`), `ProfileSchema+RateCaches.swift` for the rate-cache body shapes, and `ProfileSchema+RateCacheWithoutRowid.swift` for the meta-table rebuilds. Any column added since this plan was written must come along.

**Notes on the rate-cache columns.** `crypto_price` uses `price_usd REAL`, not `price`. `stock_price` uses `price REAL`. `exchange_rate.rate REAL`. PK on `exchange_rate` is `(base, date, quote)` (date is the second column, not the third) — this matches the leading-column scan rationale below.

import Foundation
import GRDB

/// App-level holder for the shared instrument registry, price-cache
/// services, search service, discovery service, and the
/// `SharedRegistryStore`. One instance per app; injected into every
/// `ProfileSession` so all profiles see the same registry data and
/// share price-cache rows.
///
/// Isolation: `@MainActor`. UI consumers (Settings, sidebar widgets,
/// sync stores) are all `@MainActor`-isolated; the actor types this
/// class owns are accessed via `await` and are themselves `Sendable`.
/// `@MainActor` confinement avoids the `@unchecked Sendable` carve-out
/// that would otherwise be required.
///
/// See `plans/2026-05-09-shared-instrument-registry-design.md`
/// §"DI / ownership" for the ownership rationale.
@MainActor
final class SharedInstrumentScope {
  let instrumentRegistry: any InstrumentRegistryRepository
  let cryptoPriceService: CryptoPriceService
  let stockPriceService: StockPriceService
  let exchangeRateService: ExchangeRateService
  let instrumentSearchService: InstrumentSearchService
  let cryptoTokenDiscovery: CryptoTokenDiscoveryService
  let registryStore: SharedRegistryStore

  init(
    profileIndexDatabase: any DatabaseWriter,
    coinGeckoApiKey: String?
  ) {
    let registry = GRDBInstrumentRegistryRepository(
      database: profileIndexDatabase,
      onRecordChanged: { _ in },  // wired by SyncCoordinator+ProfileIndexHooks
      onRecordDeleted: { _ in })
    self.instrumentRegistry = registry

    let cryptoPrice = CryptoPriceService(
      client: CoinGeckoClient(apiKey: coinGeckoApiKey ?? ""),
      database: profileIndexDatabase)
    self.cryptoPriceService = cryptoPrice
    self.stockPriceService = StockPriceService(
      client: YahooFinancePriceClient(),
      database: profileIndexDatabase)
    self.exchangeRateService = ExchangeRateService(
      client: FrankfurterClient(),
      database: profileIndexDatabase)

    self.instrumentSearchService = InstrumentSearchService(
      registry: registry,
      catalog: nil,           // wired separately when CoinGecko catalog is available
      resolutionClient: CompositeTokenResolutionClient(
        coinGeckoApiKey: coinGeckoApiKey ?? ""),
      stockSearchClient: YahooFinanceStockSearchClient())

    self.cryptoTokenDiscovery = CryptoTokenDiscoveryService(
      registry: registry,
      cryptoPriceService: cryptoPrice)

    self.registryStore = SharedRegistryStore(registry: registry)
  }
}
```

import Foundation
import Observation

/// App-level @Observable @MainActor store for the shared registry data.
/// Owns the data side of the old `CryptoTokenStore`. Per-session UI
/// state (isLoading/error) lives in `SettingsCryptoStore`.
///
/// Subscribes once to `registry.observeChanges()` for its lifetime; the
/// observation Task is cancelled in `deinit`.
@MainActor
@Observable
final class SharedRegistryStore {
  private(set) var registrations: [CryptoRegistration] = []
  private(set) var instruments: [Instrument] = []
  private(set) var providerMappings: [String: CryptoProviderMapping] = [:]
  private(set) var registrationsVersion: Int = 0

  private let registry: any InstrumentRegistryRepository
  private var observationTask: Task<Void, Never>?

  init(registry: any InstrumentRegistryRepository) {
    self.registry = registry
    let stream = registry.observeChanges()
    // [weak self] is required: the store stores the Task in
    // observationTask, and the Task closure captures self — strong
    // capture would create a retain cycle that prevents deinit from
    // running, which would prevent the cancel-in-deinit and leak the
    // observation forever. The @MainActor attribute and [weak self]
    // both apply (matches the established
    // SyncCoordinator+ProfileIndexHooks.swift:26,31 pattern). When the
    // store is deinitialised the for-await observes self == nil on the
    // next iteration and exits cleanly; the AsyncStream's continuation
    // is also terminated by the deinit-driven cancel.
    self.observationTask = Task { @MainActor [weak self] in
      for await _ in stream {
        await self?.loadRegistrations()
      }
    }
  }

  deinit {
    observationTask?.cancel()
  }

  private let logger = Logger(
    subsystem: "com.moolah.app", category: "SharedRegistryStore")

  func loadRegistrations() async {
    do {
      let regs = try await registry.allCryptoRegistrations()
      let all = try await registry.all()
      self.registrations = regs
      self.instruments = all
      self.providerMappings = Dictionary(
        uniqueKeysWithValues: regs.map { ($0.id, $0.mapping) })
      self.registrationsVersion += 1
    } catch {
      // The observation-driven reload path has no UI caller to surface
      // the error to. Log it (per CONCURRENCY_GUIDE §8 — fire-and-forget
      // tasks must not silently swallow errors) and leave the previous
      // data in place. UI-driven mutations route via SettingsCryptoStore
      // which catches and surfaces errors there.
      logger.error("Failed to reload registry: \(error, privacy: .public)")
    }
  }

  func setStatus(_ status: TokenPricingStatus, forId id: String) async throws {
    guard var registration = try await registry.cryptoRegistration(byId: id) else {
      throw BackendError.notFound("instrument \(id)")
    }
    registration.pricingStatus = status
    try await registry.update(registration)
    await loadRegistrations()
  }

- [ ] **Step 11.3: Implement the runner — fetch-then-write per profile.**

**Critical concurrency shape.** Each per-profile DB is opened, read, closed, and **its data is then written to the shared DB in a separate `await sharedQueue.write { … }` call**. Per-profile data is materialised into Swift value types (`[InstrumentRow]`, etc.) before the shared write begins. Do **not** nest `perProfile.read { }` inside `sharedQueue.write { }` — `Database` handles are not `Sendable` and must not escape their closure context, and a failed per-profile read inside an outer shared write would roll back previously merged profiles.

The runner is `nonisolated async` and uses the **async** `await database.write { }` overload (not the blocking sync one). Each per-profile cycle is independent: a corrupt or locked per-profile DB logs and skips, leaving previously merged profiles intact in the shared DB.

`App/SharedInstrumentScope+UnionRunner.swift`:

```swift
import Foundation
import GRDB
import OSLog

/// One-shot data-union runner that walks every per-profile DB and
/// merges its instrument + price-cache rows into the shared tables on
/// the profile-index DB. Gated by a UserDefaults flag; once true, the
/// runner is a no-op forever.
extension SharedInstrumentScope {
  static let unionFlagKey =
    "com.moolah.migration.shared-registry-union.v1.completed"

  /// Runs the union once per app install. Calling repeatedly after the
  /// flag is set is a no-op.
  ///
  /// **Concurrency.** `nonisolated async` so DB I/O is serialized by
  /// GRDB's serial executor (off the main thread). The caller awaits
  /// completion on `@MainActor` before opening any session.
  ///
  /// **Failure policy.** A per-profile DB that fails to open is logged
  /// and skipped. The remaining profiles still merge. Skipped profiles'
  /// instrument rows are absent from the shared registry until manually
  /// re-added.
  static func runUnionMigration(
    sharedQueue: any DatabaseWriter,
    profileIds: [UUID],
    fileManager: FileManager = .default,
    defaults: UserDefaults = .standard,
    logger: Logger = Logger(subsystem: "com.moolah.app", category: "SharedRegistryUnion")
  ) async {
    if defaults.bool(forKey: unionFlagKey) { return }

    // Sort ascending — defines the migration's deterministic
    // tie-breaker: the *later*-iterated profile (higher uuid) wins on
    // non-null conflicts.
    let sorted = profileIds.sorted { $0.uuidString < $1.uuidString }

    var anyProfileSucceeded = false
    for profileId in sorted {
      let url = ProfileSession.profileDatabaseDirectory(for: profileId)
        .appendingPathComponent("data.sqlite")
      guard fileManager.fileExists(atPath: url.path) else {
        logger.warning("Profile \(profileId) has no data.sqlite — skipping")
        continue
      }

      // 1. Read per-profile rows into Swift value types.
      let snapshot: PerProfileSnapshot
      do {
        let perProfile = try ProfileDatabase.open(at: url)
        snapshot = try await perProfile.read { perDB in
          try PerProfileSnapshot(reading: perDB)
        }
      } catch {
        logger.error(
          "Profile \(profileId) read failed: \(error, privacy: .public) — skipping")
        continue
      }

      // 2. Write into the shared DB in its own transaction.
      // A per-profile failure no longer rolls back previous profiles.
      do {
        try await sharedQueue.write { sharedDB in
          try mergeInstruments(into: sharedDB, snapshot: snapshot, logger: logger)
          try mergePriceCaches(into: sharedDB, snapshot: snapshot, logger: logger)
        }
        anyProfileSucceeded = true
      } catch {
        logger.error(
          "Profile \(profileId) merge failed: \(error, privacy: .public) — skipping")
      }
    }

    // Set the flag only after attempting every profile. Skipped
    // profiles are accepted as a known limitation (per the spec).
    defaults.set(true, forKey: unionFlagKey)
    logger.info(
      "Shared-registry union complete (anyProfileSucceeded=\(anyProfileSucceeded))")
  }

  /// Plain-Swift snapshot of one per-profile DB — no `Database` handle
  /// escapes the read closure. Sendable across actor boundaries.
  struct PerProfileSnapshot: Sendable {
    let instruments: [InstrumentRow]
    let exchangeRates: [ExchangeRateRow]
    let exchangeRateMeta: [ExchangeRateMetaRow]
    let stockPrices: [StockPriceRow]
    let stockTickerMeta: [StockTickerMetaRow]
    let cryptoPrices: [CryptoPriceRow]
    let cryptoTokenMeta: [CryptoTokenMetaRow]

    init(reading db: Database) throws {
      // Fetch all rows for each table.
      // ... uses `try Row.fetchAll(db, sql: ...)` plus the existing
      // GRDB row decoders.
    }
  }

  /// Bulk-upserts instrument rows from `snapshot` into the shared
  /// `instrument` table. Uses `db.cachedStatement(sql:)` so the
  /// prepared statement is reused across rows (per
  /// DATABASE_CODE_GUIDE §4 statement caching).
  ///
  /// **Upsert SQL:**
  ///
  /// ```sql
  /// INSERT INTO instrument (id, record_name, kind, name, ticker,
  ///   exchange, coingecko_id, binance_symbol, cryptocompare_symbol,
  ///   contract_address, chain_id, decimals, pricing_status,
  ///   encoded_system_fields)
  /// VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
  /// ON CONFLICT(id) DO UPDATE SET
  ///   record_name = excluded.record_name,  -- == id by invariant
  ///   kind = excluded.kind,
  ///   name = excluded.name,
  ///   ticker = COALESCE(excluded.ticker, ticker),
  ///   exchange = COALESCE(excluded.exchange, exchange),
  ///   coingecko_id = COALESCE(excluded.coingecko_id, coingecko_id),
  ///   binance_symbol = COALESCE(excluded.binance_symbol, binance_symbol),
  ///   cryptocompare_symbol = COALESCE(excluded.cryptocompare_symbol, cryptocompare_symbol),
  ///   contract_address = COALESCE(excluded.contract_address, contract_address),
  ///   chain_id = COALESCE(excluded.chain_id, chain_id),
  ///   decimals = COALESCE(excluded.decimals, decimals),
  ///   pricing_status = CASE
  ///     WHEN pricing_status = 'spam' OR excluded.pricing_status = 'spam' THEN 'spam'
  ///     WHEN pricing_status = 'priced' OR excluded.pricing_status = 'priced' THEN 'priced'
  ///     ELSE 'unpriced'
  ///   END,
  ///   encoded_system_fields = excluded.encoded_system_fields;
  /// ```
  ///
  /// The `record_name UNIQUE` constraint never fires independently
  /// because `id == record_name` is invariant for `InstrumentRecord`
  /// (asserted by `RegistryInvariantTests` in Task 17). The
  /// `ON CONFLICT(id)` clause therefore handles all collisions.
  ///

    // One BackendProvider, one SharedInstrumentScope, two ProfileSessions
    // pointing at the same registry.
    let queue = try DatabaseQueue()
    try ProfileIndexSchema.migrator.migrate(queue)
    let scope = SharedInstrumentScope(
      profileIndexDatabase: queue, coinGeckoApiKey: nil)
    try await scope.instrumentRegistry.registerCrypto(...)

    // CRITICAL: subscribe BEFORE mutating.
    let stream = scope.instrumentRegistry.observeChanges()

    // Use AsyncStream.prefix(1) — bounded by construction; consumes
    // exactly one element and the task returns. Wrap in a child Task
    // that races against a deadline arm so the test cannot hang if
    // the propagation breaks.
    let waitTask = Task {
      for await _ in stream.prefix(1) { /* consume one */ }
      return true
    }

    // Profile A's session marks spam — synchronous on @MainActor up
    // through the injected onInstrumentRemoteChange closure body, which
    // hops back to MainActor via Task to invoke notifyExternalChange().
    try await scope.registryStore.setStatus(.spam, forId: "btc")

    // Race the wait against a bounded deadline. If the propagation
    // breaks, the deadline arm cancels the wait so the test fails
    // with a deterministic `false` rather than hanging forever.
    let result = await withTaskGroup(of: Bool.self) { group in
      group.addTask { await waitTask.value }
      group.addTask {
        // Deadline: 500ms is well above the cooperative-scheduler
        // turn-around for an in-process @MainActor signal but bounded
        // for CI. The plan's "no Task.sleep" rule applies to test
        // patterns that wait for something *to happen*; here Task.sleep
        // is the deadline-cancellation arm itself, not the synchronisation
        // primitive — analogous to the bounded deadline pattern used in
        // existing CKSyncEngine integration tests.
        try? await Task.sleep(for: .milliseconds(500))
        waitTask.cancel()
        return false
      }
      // First task to complete wins; cancel the other.
      let first = await group.next() ?? false
      group.cancelAll()
      return first
    }
    #expect(result, "registry.observeChanges() did not fire after setStatus(.spam)")
