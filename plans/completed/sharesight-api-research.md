# Sharesight API — Research

## Goal

Evaluate the Sharesight API as a data source for importing Australian stock holdings and trade history into the app.

## Verdict

**Not recommended as a primary integration path.** The API is well-designed and covers our needs, but requires a paid Sharesight subscription (Investor plan, ~$29 AUD/mo) and manual credential provisioning via email. This limits the user base to existing Sharesight subscribers. CSV import (see `csv-import-design.md`) is the better first option since it works for any SelfWealth user with zero external dependencies.

Could be revisited as a "phase 2" live-sync option for users who already pay for Sharesight.

---

## API Overview

Sharesight has an official public REST API returning JSON over HTTPS. Two API versions coexist — V3 for portfolios/holdings, V2 for trades/dividends/reports.

- **V2 docs:** https://portfolio.sharesight.com/api/2/doc/index.html
- **V3 docs:** https://portfolio.sharesight.com/api/3/overview
- **Base URLs:** `https://api.sharesight.com/api/v2`, `https://api.sharesight.com/api/v3`

---

## Authentication

OAuth 2.0 exclusively — no API keys.

- **Authorization Code** grant for web apps (redirect flow)
- **Client Credentials** grant for personal/server use
- Access tokens expire in **30 minutes**; refresh tokens provided
- Header: `Authorization: Bearer <token>`
- **No self-serve provisioning** — must email api@sharesight.com to get `client_id` and `client_secret`

## Relevant Endpoints

### Portfolios

| Method | Endpoint | Notes |
|--------|----------|-------|
| GET | `v3/portfolios` | List all portfolios |
| GET | `v3/portfolios/{id}` | Single portfolio with `financial_year_end` |

### Current Holdings

| Method | Endpoint | Notes |
|--------|----------|-------|
| GET | `v3/portfolios/{id}/holdings` | All holdings — value, market value, cost base, gain |
| GET | `v3/holdings/{id}` | Single holding with `average_purchase_price`, `cost_base`, `values_over_time` |

### Trade History

| Method | Endpoint | Notes |
|--------|----------|-------|
| GET | `v2/portfolios/{id}/trades.json` | All trades; optional `start_date=YYYY-MM-DD` filter |
| GET | `v2/holdings/{holding_id}/trades.json` | Trades for a specific holding |

Trade fields: `transaction_date`, `trade_type`, `quantity`, `price`, `value`, `cost_base`, `brokerage`, `instrument_code`.

### Dividends

| Method | Endpoint | Notes |
|--------|----------|-------|
| GET | `v2/portfolios/{id}/payouts` | Dividend history with franking credits |
| GET | `v3/portfolios/{id}/income_report` | Aggregate income report |

Payout fields: `amount`, `paid_on`, `ex_date`, `franked_amount`, `unfranked_amount`.

### Other Useful Endpoints

| Method | Endpoint | Notes |
|--------|----------|-------|
| GET | `v3/portfolios/{id}/performance` | Full P&L (heavy — 3 concurrent request limit) |
| GET | `v2/portfolios/{id}/diversity` | Breakdown by market/sector |
| GET | `v2/cash_accounts/{id}/cash_account_transactions` | Cash movements |

---

## Rate Limits

- **360 requests/minute** per application
- **3 concurrent requests** max for heavy endpoints (performance, diversity, valuation reports)
- **1,500 trades per holding** hard cap
- ~3,000 trades per portfolio soft performance threshold
- Repeated invalid tokens trigger a **10-minute lockout**
- No webhook support — polling required

## Plan Requirements

API access requires a **paid Sharesight subscription**:

| Plan | Price (AUD/mo, annual) | Holdings | Portfolios | API |
|------|----------------------|----------|------------|-----|
| Starter | ~$19 | 30 | 1 | No |
| Investor | ~$29 | Unlimited | 3 | Yes |
| Expert | ~$49 | Unlimited | 5 | Yes |

Some endpoints (trades, contributions, income details) may be further gated by plan tier.

## Existing Libraries

No official SDK. Community projects:

| Project | Language | Notes |
|---------|----------|-------|
| sharesight-types | Rust | Autogenerated from Swagger spec; most complete type coverage |
| sharesight-mcp | TypeScript | MCP server with 27 V3 tools |
| HomeAssistant-Sharesight | Python | Home Assistant integration |
| sharesight-python | Python | CLI for portfolio sync |

**No Swift SDK exists.** A Swagger/OpenAPI spec may be obtainable from Sharesight, which could be used for Swift codegen.

## Comparison with CSV Import

| Factor | Sharesight API | CSV Import |
|--------|---------------|------------|
| Setup friction | High (paid plan + email for creds + OAuth) | Low (export file, import) |
| Data freshness | Live | Point-in-time snapshot |
| Automation | Possible (poll/refresh) | Manual re-import |
| Broker coverage | Any broker synced to Sharesight | Per-broker parser needed |
| User base | Only Sharesight subscribers | Any broker user |
| Implementation | Medium (OAuth + API client) | Medium (CSV parsing + UI) |
| Maintenance | API versioning, token refresh | CSV format changes |
